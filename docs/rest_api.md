# Handoff Generated API

The Handoff build writes static files into **`public/api`**. The Next.js docs app serves these at `/api/...` (or under an optional base path). This document describes the **API generated by Handoff**—not the separate Platform CLI API used for authentication and sync.

## OpenAPI specification

Request/response shapes and route details:

- **[rest.yml](./rest.yml)** — OpenAPI 3.0 spec for all Handoff-generated API routes.

Use it with Swagger UI or Redoc to browse the API.

## Base URL

- **Runtime**: The app is served from its origin; API requests use the same origin.
- **Base path**: If `HANDOFF_APP_BASE_PATH` is set, prepend it to all paths (e.g. `/handoff/api/components.json`).

All routes below are relative to that base. They are **GET-only** (static JSON, SVG, or binary files).

## Routes overview

### Components

| Path | Description |
|------|-------------|
| `GET /api/components.json` | List of all component summaries (id, title, path, properties, previews). Used by the design system menu and component list. |
| `GET /api/component/{id}.json` | Full data for one component (previews, code, page definition, properties). Used by component detail pages. |

See **[components.md](./components.md)** for the exact shape of `components.json` and `component/{id}.json`.

### Tokens

| Path | Description |
|------|-------------|
| `GET /api/tokens.json` | Full documentation object (design tokens, local styles, components metadata, assets). May be empty if no fetch has been run. |
| `GET /api/tokens/{group}.json` | One group from the documentation object (e.g. `color`, `typography`, `icons`, `logos`). |

### Assets

| Path | Description |
|------|-------------|
| `GET /api/assets/icons.json` | List of icon assets. |
| `GET /api/assets/logos.json` | List of logo assets. |
| `GET /api/assets/icons/{path}` | Raw icon file (e.g. SVG). |
| `GET /api/assets/logos/{path}` | Raw logo file. |
| `GET /api/icons-sprite.svg` | Combined SVG sprite of all icons. |
| `GET /api/icons-sprite-manifest.json` | Manifest mapping icon names to symbol IDs in the sprite. |

### Component static assets

These are also under `/api/component/` but are CSS/JS/HTML, not REST resources:

- `GET /api/component/main.css` — Shared component styles.
- `GET /api/component/{id}.css` — Component-specific styles.
- `GET /api/component/{id}.js` — Component preview script.
- `GET /api/component/{previewUrl}` — HTML preview (e.g. for iframe).

## Where the files come from

- **Components**: Written by the component pipeline ([writeComponentApi](src/transformers/preview/component/api.ts), [writeComponentSummaryAPI](src/transformers/preview/component/api.ts)) into `public/api/components.json` and `public/api/component/{id}.json`.
- **Tokens**: Written by [generateTokensApi](src/app-builder/client-config.ts) from the documentation object into `public/api/tokens.json` and `public/api/tokens/{group}.json`.
- **Assets & sprite**: Written by the documentation pipeline ([documentation.ts](src/pipeline/documentation.ts), [writeSvgSprite](src/transformers/utils/svg-sprite.ts)) into `public/api/assets/`, `public/api/icons-sprite.svg`, and `public/api/icons-sprite-manifest.json`.

At build time, `public/api` is copied into the app’s public directory and appears under `/api/` in the static export.

## Platform CLI API (separate)

The **Platform API** used by the CLI (`platform:login`, `platform:pull`, `platform:push`) is a different API (auth, projects, sync). It is not part of the Handoff-generated `public/api`. If you need to document that API, use a separate spec (e.g. `docs/platform_api.yml`).
